## Exercise 01.05 - Full Solution Walkthrough

**Module:** 01 - Build The Calculator Solution
**Associated prompt:** [1.12-implement-full-calculator-solution.prompt.md](../.github/prompts/1.12-implement-full-calculator-solution.prompt.md)

> **Optional exercise:** Complete this exercise only when you intentionally
> skip Exercises 01.01-01.04 and want Copilot to produce a complete baseline
> implementation of the calculator solution in one pass. If you completed the
> staged exercises, continue directly to
> [Exercise 02.01 - Upgrade .NET 8 To .NET 10](02.01-upgrade-dotnet-8-to-10.md).

### Learning Objectives

* Experience an end-to-end Copilot workflow that combines setup,
  implementation, refactoring, and testing in a single prompt.
* Create a complete baseline implementation when the staged Exercises
   01.01-01.04 are intentionally skipped.
* Evaluate diffs critically when Copilot generates many files at once.
* Decide when staged prompts beat one large prompt, and vice versa.

### Overview Of The Prompt

The `1.12` prompt implements the full calculator functionality described in
the [calculator PRD](../docs/prd-csharp-basic-calculator-solution.md) in one
pass. It is an optional substitute for the staged `1.12.1`-`1.12.4` prompts,
not a follow-up to them. Use it when you need a complete baseline solution
quickly or when a facilitator intentionally skips the staged build-out.

### Steps

1. Optional: if you are repeating the lab, reset the workspace first using
   the `reset-calculator-lab` skill described in
   [Exercise 99.09 - Reset Azure And Local Environments](99.09.reset-environments.md).
2. Confirm you are intentionally skipping Exercises 01.01-01.04 for this lab
   run. Do not run this prompt over a staged implementation unless you want to
   compare or replace that work.
3. In Copilot Chat, run the `1.12` full solution prompt.
4. Review the generated solution, implementation, and tests.
5. Validate everything:

   ```bash
   dotnet build src/workspace/calculator-xunit-testing/calculator.slnx
   dotnet test src/workspace/calculator-xunit-testing/calculator.slnx --no-build
   ```

### Success Criteria

* The complete solution builds and all tests pass from a single prompt run.
* You can articulate at least two trade-offs between staged and single-shot
  prompt workflows.
