## Exercise 02.04 - Reset Azure And Local Environments

**Module:** 02 - Modernize And Migrate
**Associated prompts:**
[3.03-reset-azure-environment.prompt.md](../.github/prompts/3.03-reset-azure-environment.prompt.md)
and
[3.04-reset-local-docker-pg.prompt.md](../.github/prompts/3.04-reset-local-docker-pg.prompt.md)

### Learning Objectives

* Practice safe, reviewable cleanup of cloud resources created during labs.
* Remove local Docker containers used for manual PostgreSQL workflows.
* Understand why lab environments should end in a known clean state.
* Learn to confirm destructive operations before executing them.

### Overview Of The Prompts

The `3.03` prompt cleans up Azure resources and identities created during the
migration assessment activities, and the `3.04` prompt stops and removes the
local PostgreSQL Docker container used by the manual test data workflow.
Neither prompt affects the Testcontainers-based tests, which create and remove
their own temporary containers automatically.

### Steps

1. Review what was created during [Exercise 02.03](02.03-azure-migration-assessment.md).
2. In Copilot Chat, run the `3.03` prompt and confirm each resource deletion
   before it executes.
3. Run the `3.04` prompt to remove the local PostgreSQL container, if you
   created one during manual workflows.
4. Verify cleanup:

   ```bash
   docker ps --all
   ```

### Success Criteria

* No lab-created Azure resources or identities remain.
* No leftover local PostgreSQL lab container is running or stopped.
* `dotnet test` still passes because Testcontainers manages its own
  containers.
