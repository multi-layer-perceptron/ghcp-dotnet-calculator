## Exercise 01.04 - Testing Strategy

**Module:** 01 - Build The Calculator Solution
**Associated prompt:** [1.12.4-testing-strategy.prompt.md](../.github/prompts/1.12.4-testing-strategy.prompt.md)

### Learning Objectives

* Write xUnit facts and theories for each calculator operation.
* Cover happy paths, boundary values, and error conditions such as
  divide-by-zero.
* Use parameterized tests to keep coverage broad and code small.
* Interpret `dotnet test` output and fix failing cases iteratively.

### Overview Of The Prompt

The `1.12.4` prompt defines the testing strategy for the calculator: which
behaviors to test, how to name tests using the
`[MethodName]_[Scenario]_[ExpectedBehavior]` convention, and how to structure
positive, negative, and edge-case coverage in `calculator.tests`.

### Steps

1. Complete [Exercise 01.03](01.03-refactoring-steps.md) first.
2. In Copilot Chat, run `/1.12.4-testing-strategy`.
3. Review the generated tests in `CalculatorTest.cs`.
4. Run the suite and confirm all tests pass:

   ```bash
   dotnet test src/workspace/calculator-xunit-testing/calculator.slnx
   ```

### Success Criteria

* Every arithmetic operation has passing positive and negative test cases.
* Divide and modulo by zero are asserted to throw `DivideByZeroException`.
* Test names clearly describe scenario and expected behavior.
