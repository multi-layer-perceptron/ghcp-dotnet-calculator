## Educational Repository Scope

This repository is an educational lab, not a supported production product. The
autocloudarc maintainers accept responsible vulnerability reports, but do not
claim supported versions, guaranteed remediation, a response-time commitment,
or a bug bounty.

## Report A Vulnerability Privately

Use the repository's
[private vulnerability-reporting form](https://github.com/multi-layer-perceptron/ghcp-dotnet-calculator/security/advisories/new).
Include the affected path or component, potential impact, reproduction
conditions, and the minimum evidence needed to investigate safely. If private
reporting is unavailable, use a private contact method published by the
[repository owner](https://github.com/multi-layer-perceptron).

Do not open a public Issue or Discussion for suspected vulnerabilities. Do not
send real credentials, personal data, confidential data, destructive payloads,
or unnecessary exploit material.

## Coordinated Disclosure

Allow maintainers a reasonable opportunity to investigate before public
disclosure. Maintainers may ask for clarification, coordinate a correction, or
determine that a report concerns sample code outside the project's educational
scope. Communication and remediation are best effort and are not an SLA.

## Outside Scope

- Production deployments and derivative systems not maintained by this repository
- Third-party services, extensions, packages, platforms, and their availability
- Exposed credentials or cloud resources created in a user's own account
- General support questions and non-security defects

Use [SUPPORT.md](SUPPORT.md) for support routing and
[Issues](https://github.com/multi-layer-perceptron/ghcp-dotnet-calculator/issues)
for ordinary defects.
