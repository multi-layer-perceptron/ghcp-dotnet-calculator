## Summary

This release establishes the Codespaces-ready baseline for the calculator
workshop and GitHub Actions workflow exercises. It adds a repository-managed
development container, a lifecycle guide, and exercise guidance that helps
learners start from a fork and complete the workshop in GitHub Codespaces.

## Added

* Repository-managed devcontainer configuration for GitHub Codespaces
* Codespaces lifecycle guide for setup, tool validation, port forwarding,
  prebuilds, cleanup, and work preservation
* Docker-in-Docker support for Testcontainers-backed calculator tests
* Tooling coverage for .NET, Docker, Node.js, Python, PowerShell, GitHub CLI,
  Azure CLI, Azure Developer CLI, Bicep, Terraform, Playwright, and MCP usage
* Codespaces guidance for the main calculator labs and GitHub Actions workflow
  exercises

## Changed

* Main README and exercise guidance now use a fork-first Codespaces workflow
* Blazor local run guidance now binds to `http://0.0.0.0:5000` for Codespaces
  port forwarding
* Playwright MCP configuration now uses pinned, headless execution
* GitHub agentic workflow guidance now distinguishes preferred
  `copilot-requests: write` permissions from short-lived token fallback use
* Cleanup guidance now requires work preservation before deleting a Codespace

## Fixed

* Added .NET 8 runtime coverage for staged pre-upgrade exercises while keeping
  the active workspace aligned with .NET 10
* Removed unused pnpm installation from the Codespaces baseline
* Moved `gh-aw` installation into post-create validation so it installs in the
  learner user context
* Reduced Python setup cost by using the operating-system-provided Python
* Added bounded retry behavior for network-dependent post-create setup steps

## Validation

* Devcontainer configuration resolved successfully with the Dev Containers CLI
* Docker Desktop and Docker Engine were verified locally before validation
* Devcontainer feature layers built successfully
* Docker-in-Docker startup reached during container validation
* `gh-aw` installed successfully during post-create validation
* Static JSON, Bash, PowerShell, and focused Markdown checks passed for the
  changed areas
* `git diff --check` passed for the focused instruction update

## Known Limitations

* Local Docker Desktop validation on Windows was blocked during npm and
  Playwright package download by Windows Defender TLS interception. This was
  accepted as a host-specific limitation and is not expected to affect
  GitHub-hosted Codespaces.
* Full Testcontainers-backed calculator test execution inside the final
  devcontainer should be rerun in a clean Codespaces environment or another
  network path where npm registry access is not intercepted.

## Upgrade Notes

* Learners should create Codespaces from their own fork before starting the lab
  or workflow exercises.
* Existing local users can continue working locally, but Codespaces is now the
  preferred low-friction workshop path.
* Maintainers should use milestone `M1 - Codespaces-ready workshop baseline`,
  tag `v0.2.0`, and release title
  `v0.2.0 - Codespaces-ready workshop baseline` for this milestone.
