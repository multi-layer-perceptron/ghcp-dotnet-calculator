### 99.03.1 Examine Current MCP Configuration

MCP servers extend Copilot with tools outside the repository. They can retrieve
official documentation, fetch web content, query internal systems, or persist
project knowledge depending on the server you configure.

Check whether this repository already has a local MCP configuration:

```powershell
Test-Path .copilot/mcp-config.json
```

For the current repository, this should return `False`. That means the
calculator exercises work without MCP and any configuration you create in this
lab is optional.

If the file exists in your workspace, inspect it before changing it:

```powershell
Get-Content .copilot/mcp-config.json
```

### 99.03.2 Create A Practice MCP Configuration

Create a `.copilot` folder and a minimal MCP configuration for documentation and
web retrieval workflows:

```powershell
New-Item -ItemType Directory -Path .copilot -Force | Out-Null

@'
{
  "mcpServers": {
    "microsoft-learn": {
      "type": "http",
      "url": "https://learn.microsoft.com/api/mcp",
      "tools": ["*"]
    },
    "fetch": {
      "type": "local",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-fetch"],
      "tools": ["*"]
    }
  }
}
'@ | Out-File -FilePath .copilot/mcp-config.json -Encoding utf8
```

Verify the file:

```powershell
Get-Content .copilot/mcp-config.json
```

The `microsoft-learn` server is useful for official Microsoft documentation
related to .NET, Blazor, Azure App Service, Azure Developer CLI, Bicep, and
GitHub Actions. The `fetch` server is useful when you need to retrieve public
web pages or raw documentation content during a lab.

> [!IMPORTANT]
> Treat MCP configuration as executable tool configuration. Review server
> packages, commands, URLs, and exposed tools before committing the file.

### 99.03.3 Understand Server Types

This practice file uses two common MCP server styles.

| Server | Type | Why it is useful in this repo |
| ------ | ---- | ----------------------------- |
| `microsoft-learn` | `http` | Looks up official Microsoft documentation for .NET, Blazor, Azure App Service, Bicep, and Azure Developer CLI tasks |
| `fetch` | `local` | Runs through `npx` and retrieves public web or raw documentation content when a lab references an external page |

HTTP servers connect to a remote endpoint. Local servers run a command on your
machine and communicate through standard input and output. Local servers require
the runtime named in the command. In this example, `fetch` requires Node.js and
`npx`.

Check whether `npx` is available before relying on the local server:

```powershell
Get-Command npx -ErrorAction SilentlyContinue
```

If the command returns no result, install Node.js or remove the `fetch` entry
from the practice configuration.

### 99.03.5 Connect MCP Usage To This Project

Use MCP only when it adds information that the repository itself does not have.

Good fits for this calculator lab:

* Checking current Azure App Service guidance before changing `infra/main.bicep`
  or `azure.yaml`.
* Looking up current Blazor hosting or component guidance before changing
  `calculator.web`.
* Fetching external lab references or raw README files when creating new
  exercises.
* Comparing Microsoft Learn deployment guidance with the repository's existing
  Azure migration assessment.

Poor fits:

* Replacing local code search for files already in this repository.
* Storing secrets, local connection strings, or personal credentials.
* Adding broad tool access without a concrete lab need.
* Committing practice MCP configuration without team review.

### 99.03.6 Clean Up Or Keep The Configuration

Decide whether `.copilot/mcp-config.json` belongs in this repository.

To remove the practice configuration:

```powershell
Remove-Item .copilot/mcp-config.json -ErrorAction SilentlyContinue
Remove-Item .copilot -Recurse -Force -ErrorAction SilentlyContinue
```

Keep it only if the team wants a durable MCP setup for future participants and
has reviewed the server commands, URLs, and exposed tools.

### Key Takeaways

| Concept | Purpose |
| ------- | ------- |
| MCP server | Adds external tools to Copilot beyond local repository context |
| HTTP server | Connects to a remote MCP endpoint without a local process |
| Local server | Runs a command on your machine and exposes tools over standard input and output |
| Tool allowlist | Controls which server tools are available to Copilot |
| Cleanup step | Prevents temporary tool configuration from becoming accidental repo policy |

Best practices:

* Start without MCP when local files answer the question.
* Add MCP servers only for specific documentation, retrieval, or integration
  needs.
* Review local server commands before running them.
* Avoid storing secrets or credentials in MCP configuration.
* Restart Copilot sessions after changing MCP configuration.
