## Exercise 03.02 - Comprehensive Quality Gate

**Module:** 03 - Quality, Security, And Wrap-Up
**Associated prompt:** [12.00.test-for-quality.prompt.md](../.github/prompts/12.00.test-for-quality.prompt.md)
**Associated agent:** [software-quality-tester](../.github/agents/software-quality-tester.agent.md)

### Learning Objectives

* Execute a comprehensive QA workflow covering automated test generation,
  edge case analysis, regression testing, and coverage improvement.
* Use a custom Copilot agent as a QA persona that plans scope, runs checks,
  and reports findings by severity.
* Reconcile a large legacy prompt checklist against the live repository
  state before acting on it.
* Produce a QA report with pass/fail status, residual risk, and next steps.

### Overview Of The Prompt

The `12.00` prompt is a comprehensive QA requirements catalog with phased
checks spanning functional testing, edge cases, test data, bug reproduction,
regression, and automation. It is intentionally broad and contains some
legacy assumptions (for example, .NET 8 targets and older test counts). The
`software-quality-tester` agent uses this prompt as its context source and is
responsible for translating it into a current, repository-specific test plan.

### Steps

1. Complete through
  [Exercise 02.03 - Azure Migration Assessment](02.03-azure-migration-assessment.md)
  so the quality gate has a realistic target.
2. Switch to the `software-quality-tester` agent in Copilot Chat.
3. Run the `12.00` test-for-quality prompt.
4. Review the proposed QA scope before allowing broad checks to run.
5. Examine the final report and address high-severity findings first.

### Success Criteria

* The full test suite passes: `dotnet test src/workspace/calculator-xunit-testing/calculator.slnx`.
* The QA report distinguishes current findings from legacy prompt
  assumptions.
* High-severity findings are fixed or explicitly accepted with rationale.
