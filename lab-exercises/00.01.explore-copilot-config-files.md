## Exercise 00.01 - Explore Copilot Configuration Files

**Module:** 00 - Explore The Copilot Workspace
**Associated prompt:** None. This is an orientation exercise.

In this lab you will discover and understand the Copilot configuration files
that ship with this calculator repository before using them in the staged
exercises.

> [!NOTE]
> Presenter pace: 4 minutes. Self-paced: 10 minutes.

References:

* [Custom agents configuration](https://docs.github.com/en/copilot/reference/custom-agents-configuration)
* [Agent skills](https://docs.github.com/en/copilot/using-github-copilot/using-copilot-agent-skills)
* [Custom instructions](https://docs.github.com/en/copilot/customizing-copilot/adding-custom-instructions-for-github-copilot)
* [Prompt files](https://docs.github.com/en/copilot/using-github-copilot/using-prompt-files)

### Learning Objectives

* Identify the Copilot customization files used by this repository.
* Understand how repo-wide instructions, path instructions, skills, agents, and
  prompts work together.
* Practice reading YAML frontmatter and Markdown bodies in Copilot assets.
* Use the configuration map to choose the right prompt or agent for later lab
  exercises.

### 00.01.1 Optional Marketplace Discovery


The rest of this exercise works entirely from files already committed to this
repository.

### 00.01.2 Configuration Ecosystem

This repository contains a focused Copilot configuration ecosystem for a .NET
calculator learning workspace.

| Location | Purpose | Current count | Used in |
| -------- | ------- | ------------- | ------- |
| `.github/agents/` | Specialized AI personas in `.agent.md` files | 15 | Development, testing, security, DevOps, data, and review flows |
| `.github/skills/` | Auto-activating knowledge packs with `SKILL.md` files | 5 | Repeatable calculator setup, CSV, PostgreSQL, README, and Blazor workflows |
| `.github/prompts/` | Reusable command templates in `.prompt.md` files | 46 | Staged calculator, modernization, Azure, security, and workflow exercises |
| `.github/instructions/` | Scoped instruction files for reviews, docs, testing, workflows, and repo patterns | 11 | Always considered when matching files are in context |
| `.github/copilot-instructions.md` | Repository-wide Copilot guidance | 1 | Every Copilot interaction in this repo |
| `.github/workflows/` | GitHub Actions examples for later workflow labs | 24 | CI/CD and GitHub Actions practice |

This repository does not currently include root `AGENTS.md`, `.github/hooks/`,
or `.copilot/mcp-config.json` files. If those appear later, treat them as
additional Copilot or automation configuration surfaces and inspect them before
changing behavior.

Quick look in PowerShell:

```powershell
Get-ChildItem .github/agents -Filter *.agent.md
Get-Content .github/agents/sw-developer.agent.md

Get-ChildItem .github/skills -Directory
Get-Content .github/skills/calculator-setup/SKILL.md

Get-ChildItem .github/prompts -Filter *.prompt.md
Get-Content .github/prompts/1.12.1-solution-setup.prompt.md
```

Quick look in Bash:

```bash
ls .github/agents/*.agent.md

find .github/skills -maxdepth 2 -name SKILL.md
sed -n '1,80p' .github/skills/calculator-setup/SKILL.md

```

### 00.01.3 Instruction Hierarchy

Copilot configuration loads in layers. The important idea is that instructions
stack rather than replace each other.

| Priority | Source | When loaded |
| -------- | ------ | ----------- |
| 1 | `.github/copilot-instructions.md` | Every interaction in this repository |
| 2 | `.github/instructions/*.instructions.md` and `.instruction.md` | When the `applyTo` pattern or file context matches |
| 3 | `.github/skills/*/SKILL.md` | When the skill description matches the task |
| 4 | `.github/agents/*.agent.md` | When an agent is explicitly selected or invoked |
| 5 | `.github/prompts/*.prompt.md` | When you run a reusable prompt from Copilot Chat |

For example, when you ask Copilot to follow
`1.12.1-solution-setup.prompt.md`, the prompt delegates to the
`calculator-setup` skill. The repo-wide instructions still apply, and any
matching PowerShell, Markdown, or C# instructions can also apply depending on
the files being edited.

### 00.01.4 How Copilot Finds This Codebase

Copilot can use several layers of repository intelligence when answering
questions or making edits.

| Layer | How it works | Typical scope |
| ----- | ------------ | ------------- |
| Semantic indexing | Builds a meaning-based index of repository code and documentation | Local workspace |
| Language server integration | Uses C#, PowerShell, Markdown, YAML, and other language services for diagnostics and symbol navigation | Open workspace |
| File and text search | Uses fast file and text search for exact names, prompt IDs, and paths | Local workspace |
| GitHub tools | Search and inspect remote GitHub repositories when available and authorized | Remote repositories |

When you ask, "How does the calculator test data flow work?", Copilot can use
the prompt files, skills, test project, CSV file, PostgreSQL test code, and
repository instructions together.

### 00.01.5 Try It

Open Copilot Chat from the repository root and try these prompts:

```text
What Copilot prompt should I run first in this calculator workspace?
```

```text
Which skill supports the 1.12.1 solution setup prompt, and what files does it create?
```

```text
Explain the difference between .github/copilot-instructions.md, .github/instructions, .github/skills, and .github/prompts in this repo.
```

Then inspect the answer against the files you read in this exercise. The goal is
to confirm that Copilot is grounding its response in the repository instead of
answering from generic memory.

### Success Criteria

* You can identify where agents, prompts, skills, instructions, and workflows
  live in this repository.
* You can explain why `1.12.1-solution-setup.prompt.md` uses the
  `calculator-setup` skill.
* You can name at least one repo-wide instruction and one path-specific
  instruction that may affect later exercises.
* You understand that this repo does not currently include `AGENTS.md`, hooks,
  or MCP configuration files.

### Key Takeaways

| Concept | File location | Purpose |
| ------- | ------------- | ------- |
| Repo-wide instructions | `.github/copilot-instructions.md` | Always-on rules for this calculator workspace |
| Path instructions | `.github/instructions/` | File-scoped guidance for reviews, docs, tests, workflows, and repo conventions |
| Agents | `.github/agents/*.agent.md` | Specialized personas such as software developer, tester, DevOps, and security roles |
| Skills | `.github/skills/*/SKILL.md` | Repeatable procedures and knowledge packs that load when relevant |
| Prompts | `.github/prompts/*.prompt.md` | Reusable commands for staged lab workflows |
| Workflows | `.github/workflows/*.yml` | GitHub Actions examples used in later automation exercises |
