## Exercise 03.03 - Final Validation And Handoff

**Module:** 03 - Quality, Security, And Wrap-Up
**Associated prompt:** None. Final teardown is deferred to Exercise 99.09.

### Learning Objectives

* Preserve final implementation, test, security, and quality evidence.
* Confirm the completed calculator workspace is ready for Module 99.
* Create a checkpoint or handoff before customization exercises begin.
* Understand why destructive cleanup belongs at the end of the complete
  exercise sequence.

### Overview

Earlier versions of this exercise invoked the `1.13` cleanup prompt and removed
the generated calculator workspace. Module 99 depends on that completed
workspace, so cleanup now occurs only in
[Exercise 99.09 - Reset Azure And Local Environments](99.09.reset-environments.md).

Use this exercise to preserve a known-good state. Do not use the
`reset-calculator-lab` skill to remove the workspace yet.

### Steps

1. Review the output from [Exercise 03.02](03.02-comprehensive-quality-gate.md)
   and resolve or record any findings you want to retain.
2. Run one final validation pass:

   ```bash
   dotnet test src/workspace/calculator-xunit-testing/calculator.slnx
   ```

3. Confirm the finished workspace required by Module 99 exists:

   ```powershell
   Test-Path src/workspace/calculator-xunit-testing/calculator.slnx
   Test-Path src/workspace/calculator-xunit-testing/calculator.library/CalculatorOperations.cs
   ```

4. Save test output, reports, screenshots, and issue references that should
   survive the remaining exercises.
5. Review `git status --short`, then create a commit, stash, patch, or session
   handoff appropriate for your workflow.

### Success Criteria

* Final test, security, and quality evidence is saved.
* The completed calculator workspace remains available.
* The working tree state is understood and recoverable.
* No Azure, Docker, source-workspace, or Copilot customization cleanup has run.
