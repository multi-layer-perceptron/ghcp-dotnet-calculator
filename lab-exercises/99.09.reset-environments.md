## Exercise 99.09 - Reset Azure And Local Environments

**Module:** 99 - Finished Project Customization
**Associated prompts:**
[3.03-reset-azure-environment.prompt.md](../.github/prompts/3.03-reset-azure-environment.prompt.md)
and
[3.04-reset-local-docker-pg.prompt.md](../.github/prompts/3.04-reset-local-docker-pg.prompt.md).
**Associated skill:** `reset-calculator-lab`. The legacy
[1.13 cleanup prompt](../.github/prompts/1.13-cleanup-solution.prompt.md)
delegates to this skill for compatibility.

### Learning Objectives

* Practice safe, reviewable cleanup of cloud resources created during labs.
* Remove local Docker containers used for manual PostgreSQL workflows.
* Remove generated workspace and test artifacts only when that reset depth is
   required.
* Understand why shared lab resources should remain available until the full
  exercise sequence is complete.
* Learn to confirm destructive operations before executing them.

### Overview Of The Prompts

The `3.03` prompt cleans up Azure resources and identities created during the
migration assessment activities, and the `3.04` prompt stops and removes the
local PostgreSQL Docker container used by the manual test data workflow.
Neither prompt affects the Testcontainers-based tests, which create and remove
their own temporary containers automatically.

The `reset-calculator-lab` skill coordinates the selected reset depth. Its
bundled PowerShell script previews, preserves, verifies, and removes the
generated workspace without embedding Azure, Docker, or Git deletion in the
same command.

Run this exercise only after completing the preceding exercises and preserving
any evidence or outputs you want to keep. Delaying resource cleanup ensures the
Azure and local PostgreSQL environments remain available for later security,
quality, hooks, orchestration, workflow, and capstone activities.

> [!CAUTION]
> The default reset removes only confirmed lab-created runtime resources. The
> generated-workspace, local Copilot-state, and Git reset options are separate
> because they can destroy work or reusable learning. Preview every target and
> preserve required evidence before using those options.

### Choose A Reset Depth

| Reset depth | Removes | Preserves |
| ----------- | ------- | --------- |
| Runtime resources (default) | Confirmed lab-created Azure resources and the named PostgreSQL container | Source files, customizations, history, and local Copilot state |
| Generated workspace | Runtime resources plus the generated calculator solution and ignored build/test output | Tracked repository content and local Copilot state |
| Full lab state | All preceding items plus selected practice edits and optional learned state | Only content explicitly backed up or committed |

Use the runtime reset unless you intend to repeat the lab from its initial
setup state. Never apply a full Git reset to a working tree you have not
reviewed and backed up.

### Steps

1. Complete [Exercise 99.08](99.08.capstone-exercises.md) and save any final
   test results, reports, screenshots, or session handoff notes you need.
2. Inspect the working tree before any cleanup:

   ```powershell
   git status --short
   git diff --stat
   ```

   Commit, stash with untracked files, or export a patch before continuing if
   the output contains work you need to keep.
3. Ask Copilot Chat to use the `reset-calculator-lab` skill and specify the
   `runtime`, `workspace`, or `full` reset depth. When you omit a depth, Copilot
   should ask you to choose one before making changes.
4. Review the Azure and local resources created throughout the exercise set.
5. In Copilot Chat, run the `3.03` prompt and confirm each resource deletion
   before it executes.
6. Run the `3.04` prompt to remove the local PostgreSQL container, if you
   created one during manual workflows.
7. Verify runtime cleanup:

   ```bash
   docker ps --all --filter "name=^/test-container$"
   ```

8. Verify the Azure cleanup results reported by the `3.03` prompt before
   closing the lab environment.

### Optional Generated-Workspace Reset

Use this reset only when you want to return to the state before the calculator
solution was generated.

1. Preview ignored files such as `bin/`, `obj/`, and test output without
   deleting them:

   ```powershell
   git clean -ndX
   ```

2. After reviewing every path, remove only ignored generated files if needed:

   ```powershell
   git clean -fdX
   ```

3. Confirm that the repository-provided `src/completed/` folder exists and is
   empty. The skill blocks removal when it cannot preserve the current run.
4. Ask Copilot to use the `reset-calculator-lab` skill with `workspace` reset
   depth. Review its preview and confirmation request.
5. To preview the bundled script directly, run:

   ```powershell
   pwsh .github/skills/reset-calculator-lab/scripts/Remove-DotnetSlnForCalculator.ps1 -WhatIf
   ```

6. After reviewing the exact paths, run the confirmed operation:

   ```powershell
   pwsh .github/skills/reset-calculator-lab/scripts/Remove-DotnetSlnForCalculator.ps1 -Confirm
   ```

> [!IMPORTANT]
> The script first copies the generated solution to
> `src/completed/calculator-xunit-testing/` and verifies that destination before
> deleting `src/workspace/calculator-xunit-testing/`. It fails without deleting
> the workspace when `src/completed/` is missing or not empty. Use the
> `calculator-setup` skill to start another run after preserving or relocating
> the completed copy.

### Optional Docker Data Cleanup

The `3.04` prompt removes the named container. Docker volumes and networks can
remain if they were created separately. Inspect them before deletion:

```powershell
docker volume ls
docker network ls
```

Remove only volumes or networks whose names and inspection details confirm
they belong exclusively to this lab. Do not use a global Docker prune command
on a development machine with unrelated workloads.

### Optional Copilot State Cleanup

Preserve `.github/`, `.mcp.json`, and `.vscode/mcp.json` when they are tracked
repository configuration. Review `git status --short` before removing any
untracked configuration created only for the lab.

Exercise 99.08 documents continuous-learning state under
`~/.copilot/homunculus/`. Keep it when you want learned instincts and
observations to persist. To reset that state, inspect the directory first and
remove it only after exporting anything you want to retain:

```powershell
Get-ChildItem ~/.copilot/homunculus -Force -Recurse
```

Memory MCP data location depends on the server configuration. Inspect that
configuration and its storage path rather than deleting a guessed directory.

### Optional Full Git Reset

The safest way to discard Module 99 practice edits is to use a short-lived
branch and switch back to the preserved starting branch. For uncommitted work,
review `git status --short`, save a patch or stash, and preview untracked files
with `git clean -nd` before considering deletion.

Do not place `git reset --hard`, `git restore .`, or `git clean -fd` in an
automated reset flow. Those commands can remove unrelated work and should be
run only after explicit review and confirmation.

### Final Verification

Confirm the selected reset depth rather than expecting every optional item to
be absent:

```powershell
git status --short
docker ps --all --filter "name=^/test-container$"
Test-Path src/workspace/calculator-xunit-testing
Test-Path src/completed/calculator-xunit-testing
```

### Success Criteria

* The complete exercise sequence is finished before shared resources are
  removed.
* No lab-created Azure resources or identities remain.
* No leftover local PostgreSQL lab container is running or stopped.
* Final evidence is preserved before cleanup.
* `dotnet test` can still use Testcontainers because it manages its own
  temporary containers.
* Optional generated files, Docker data, Copilot state, or practice edits are
   removed only when the selected reset depth requires it.
* Generated-workspace cleanup preserves the completed solution before removing
   the workspace copy.
