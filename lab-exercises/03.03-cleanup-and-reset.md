## Exercise 03.03 - Cleanup And Reset

**Module:** 03 - Quality, Security, And Wrap-Up
**Associated prompt:** [1.13-cleanup-solution.prompt.md](../.github/prompts/1.13-cleanup-solution.prompt.md)

### Learning Objectives

* Close the lab by preserving the final implementation, test, security, and
  quality evidence.
* Compare the built solution against the completed reference solution only
  after the hands-on work is finished.
* Use a prompt or script to reset the workspace to a clean state.
* Understand why lab teardown belongs after validation, not in the middle of
  the exercise sequence.

### Overview Of The Prompt

The `1.13` prompt was originally authored as the cleanup companion for the
Module 01 setup flow. In this lab sequence, it is intentionally re-indexed as
the final exercise because cleanup should happen after implementation,
modernization, security review, and comprehensive QA are complete.

Use the prompt as an end-of-lab teardown workflow. The repository also ships
ready-made scripts you can run directly:
`src/workspace/Remove-DotnetSlnForCalculator.ps1` removes the generated
workspace, and `src/workspace/Set-DotnetSlnForCalculator.ps1` recreates it if
you want to repeat the lab from the beginning.

### Steps

1. Review the output from [Exercise 03.02](03.02-comprehensive-quality-gate.md)
   and make sure any quality or security findings you want to keep are saved
   in commits, notes, or issue drafts.
2. Run one final validation pass so the lab ends with known evidence:

   ```bash
   dotnet test src/workspace/calculator-xunit-testing/calculator.slnx
   ```

3. Optional: after the lab work is complete, compare your implementation with
   the hidden reference under `src/completed/` if your facilitator makes it
   available for review. Do not use the reference solution earlier in the lab.
4. Commit or stash any work you want to keep. Cleanup removes generated
   solution files in the active workspace.
5. In Copilot Chat, run the `1.13` cleanup prompt, or run the script directly:

   ```powershell
   pwsh -NoProfile -ExecutionPolicy Bypass -File src/workspace/Remove-DotnetSlnForCalculator.ps1
   ```

6. Confirm the workspace is reset. Recreate it only if you want to repeat the
   lab from Module 01:

   ```powershell
   pwsh -NoProfile -ExecutionPolicy Bypass -File src/workspace/Set-DotnetSlnForCalculator.ps1
   ```

### Success Criteria

* Final test, security, and QA evidence is saved before teardown.
* The active workspace returns to a known clean state.
* Re-running the setup script rebuilds a working solution, proving the
   setup and cleanup pair remains repeatable for the next lab run.

### Wrap Up

This is the final exercise. After cleanup, return to the [README](../README.md)
to repeat the lab from Module 01 or explore the remaining prompt workflows.
